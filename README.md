🛠️ คู่มือสำหรับนักพัฒนา SUT FixIt (Developer Guide)
คู่มือฉบับสมบูรณ์สำหรับสมาชิกในทีม เพื่อใช้ในการดึงโค้ดไปรันในเครื่องตัวเอง ติดตั้งเครื่องมือ และขั้นตอนการส่งงาน

📌 ส่วนที่ 1: สิ่งที่ต้องมีในเครื่องก่อนเริ่มทำงาน (Prerequisites)
ก่อนจะเริ่มพิมพ์คำสั่งใดๆ ให้เช็คให้ชัวร์ว่าในคอมพิวเตอร์และมือถือของเรามีโปรแกรมเหล่านี้แล้ว:

VS Code (Visual Studio Code): โปรแกรมสำหรับใช้เขียนโค้ด

Node.js: ตัวรันคำสั่งต่างๆ ของระบบ (เช็คว่าลงหรือยังโดยพิมพ์ node -v ใน Terminal)

Git: ระบบจัดการเวอร์ชันโค้ด (เช็คว่าลงหรือยังโดยพิมพ์ git -v ใน Terminal)

แอป Expo Go: โหลดติดมือถือไว้สำหรับเทสแอป (มีทั้งใน iOS App Store และ Android Play Store)

Android Studio (ไม่บังคับ): เฉพาะสำหรับคนที่ไม่มีมือถือให้เทส และต้องการรัน "มือถือจำลอง (Emulator)" บนหน้าจอคอมพิวเตอร์

🚀 ส่วนที่ 2: ขั้นตอนการดึงโค้ดและติดตั้งระบบ (ทำแค่ครั้งแรกครั้งเดียว)
ทำตามขั้นตอนนี้ ทีละบรรทัด เพื่อเตรียมโปรเจกต์ให้พร้อมรัน

1. สร้างโฟลเดอร์จัดเก็บ
สร้างโฟลเดอร์เปล่าๆ ไว้ในเครื่อง (เช่น ในไดรฟ์ D: หรือหน้า Desktop) จากนั้นคลิกขวาที่โฟลเดอร์ เลือก "Open with Code"

2. เปิดหน้าจอพิมพ์คำสั่ง (Terminal)
ในโปรแกรม VS Code ให้ไปที่เมนูด้านบน เลือก Terminal > New Terminal

3. ดึงโค้ดจาก GitHub ของทีม
ก๊อปปี้คำสั่งนี้ไปวางแล้วกด Enter (ระบบจะสร้างโฟลเดอร์ชื่อ sut-fixit ขึ้นมาพร้อมโค้ดทั้งหมด):

Bash
git clone https://github.com/Ch1tiipat/sut-fixit.git
4. ย้ายตัวเองเข้าไปในโฟลเดอร์โปรเจกต์ (🚨 สำคัญมาก ห้ามข้ามขั้นตอนนี้!)

Bash
cd sut-fixit
5. ติดตั้งไฟล์ระบบพื้นฐาน (Dependencies)
คำสั่งนี้จะทำการดาวน์โหลดไฟล์แพ็กเกจที่ซ่อนอยู่กลับมา (อาจจะใช้เวลาโหลดสัก 1-3 นาที ขึ้นอยู่กับความเร็วเน็ต):

Bash
npm install
6. ติดตั้งไลบรารีเสริมที่แอปเราต้องใช้
ก๊อปปี้ชุดคำสั่งด้านล่างนี้ไปรันรวดเดียวได้เลย เพื่อลงเครื่องมือระบบแผนที่, ระบบเลือกรูปภาพ และระบบกราฟ:

Bash
npx expo install expo-location react-native-maps expo-image-picker expo-file-system react-native-chart-kit react-native-svg
🌐 ส่วนที่ 3: การตั้งค่า Ngrok (เชื่อมต่ออินเทอร์เน็ตให้แอป)
เนื่องจากแอปของเรามี "ระบบอัปโหลดรูปภาพเข้าฐานข้อมูล (Firebase)" การรันเซิร์ฟเวอร์แบบธรรมดาอาจทำให้ระบบเน็ตเวิร์คชนกัน จึงต้องใช้เครื่องมือที่ชื่อว่า Ngrok ช่วยทะลวงอินเทอร์เน็ต

1. ติดตั้งโปรแกรม Ngrok ลงในเครื่อง (รันแค่ครั้งเดียวในชีวิต):

Bash
npm install -g @expo/ngrok
(หมายเหตุ: ถ้าใครใช้ Mac แล้วขึ้น Error ให้เติมคำว่า sudo ไว้ข้างหน้า เป็น sudo npm install -g @expo/ngrok แล้วใส่รหัสผ่านเครื่อง)

2. ยืนยันสิทธิ์ด้วย Authtoken ของโปรเจกต์เรา:

Bash
npx ngrok config add-authtoken 3Bo9tFyL3bRqwAFomG3ZJkqRWQT_7H3J9QqQY6rSTSMi6WV6S
▶️ ส่วนที่ 4: วิธีรันแอปพลิเคชันเพื่อทดสอบการทำงาน
เมื่อโค้ดพร้อมแล้ว เลือกวิธีรันแอปตามอุปกรณ์ที่คุณถนัดได้เลย

วิธีที่ 1: รันบนมือถือจริง (แนะนำ เพราะเทสกล้องและ GPS ได้สมจริง)
ตรวจสอบให้แน่ใจว่าคุณอยู่ในโฟลเดอร์ sut-fixit

พิมพ์คำสั่งเปิดเซิร์ฟเวอร์แบบทะลวงเน็ตเวิร์ค:

Bash
npx expo start --tunnel
รอจนกว่าจะมี QR Code ปรากฏขึ้นมาบนหน้าจอคอมพิวเตอร์

หยิบมือถือขึ้นมา:

iPhone (iOS): เปิดแอป "กล้องถ่ายรูป" สแกน QR Code แล้วกดเปิดใน Expo Go

Android: เปิดแอป "Expo Go" แล้วกดปุ่ม Scan QR Code

วิธีที่ 2: รันบนมือถือจำลอง (สำหรับคนใช้ Android Studio)
เปิดโปรแกรม Android Studio ไปที่เมนู Virtual Device Manager แล้วกดปุ่ม Play (▶️) เพื่อเปิดหน้าจอมือถือจำลองขึ้นมารอไว้

กลับมาที่ VS Code พิมพ์คำสั่ง:

Bash
npx expo start --tunnel
เมื่อ QR Code ขึ้นมา ให้คลิกที่ Terminal แล้วกดปุ่มตัวอักษร a บนคีย์บอร์ด 1 ครั้ง

แอปจะเด้งไปติดตั้งและเปิดบนมือถือจำลองอัตโนมัติ (ถ้าหน้าต่างถามให้ติดตั้ง Expo Go ลงในมือถือจำลอง ให้พิมพ์ y แล้วกด Enter)

📍 ส่วนที่ 5: วิธีแก้ปัญหา GPS ในมือถือจำลองชี้ไปผิดที่
หากรันแอปในหน้าจอมือถือจำลอง แล้วพิกัด GPS ไปโผล่ที่ต่างประเทศ ให้ตั้งค่าพิกัดให้เป็น มหาวิทยาลัยเทคโนโลยีสุรนารี (มทส.) ดังนี้:

ที่เมนูด้านขวามือของหน้าจอมือถือจำลอง ให้คลิกที่ไอคอน จุดสามจุด (...)

เลือกเมนู Location แถบด้านซ้าย

ในช่อง Search ให้พิมพ์พิกัดนี้ลงไป: 14.8818, 102.0205 แล้วกด Enter

กดปุ่ม Save Location / Set Location ด้านล่าง (หมุดจะเด้งมาที่ มทส. ทันที)

📤 ส่วนที่ 6: วิธีอัปเดตและส่งงาน (Git Workflow)
เมื่อทุกคนเขียนโค้ดในส่วนของตัวเองเสร็จแล้ว ให้กด Save (Ctrl + S) ให้ครบทุกไฟล์ แล้วทำตาม 3 ขั้นตอนนี้เพื่อส่งโค้ดไปรวมกับของเพื่อนบน GitHub

🚨 กฎเหล็กก่อนส่งงาน: ก่อนจะพิมพ์คำสั่งส่งงาน ให้พิมพ์ git pull ก่อนเสมอ เพื่อดึงโค้ดล่าสุดที่เพื่อนอาจจะเพิ่งอัปเดตมาใส่เครื่องเรา ป้องกันปัญหาไฟล์ชนกัน!

เมื่อพร้อมแล้ว เปิด Terminal พิมพ์ 3 คำสั่งนี้ทีละบรรทัด:

1. เก็บไฟล์ที่แก้ไขทั้งหมดลงกล่อง (อย่าลืมเว้นวรรคและใส่จุด . ด้วย)

Bash
git add .
2. แปะป้ายบอกทีมงานว่าอัปเดตอะไรไปบ้าง (เปลี่ยนข้อความในเครื่องหมายคำพูดได้เลย)

Bash
git commit -m "ออกแบบหน้า UI การแจ้งซ่อมเสร็จแล้ว"
3. ดันโค้ดทั้งหมดขึ้นสู่ GitHub

Bash
git push
(หากขึ้นคำว่า "Everything up-to-date" แปลว่าไม่มีไฟล์ไหนเปลี่ยนแปลง แต่ถ้าเปอร์เซ็นต์วิ่งจนเสร็จ แสดงว่าเพื่อนๆ ในทีมสามารถดึงโค้ดคุณไปใช้งานต่อได้แล้ว!)
